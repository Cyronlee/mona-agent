## Overview

This feature focuses on the post-bid acceptance phase, where an order is active. It includes dashboards for viewing order details, updating statuses (for Locals), and an integrated messaging system to resolve queries, share updates, and coordinate details between Shoppers and Locals.

## Design Constraints

*   Order status updates should be simple for Locals to perform.
*   Shoppers should receive timely notifications about status changes.
*   The messaging system must be real-time or near real-time.
*   All communication related to an order should be easily accessible within that order's context.
*   Dispute resolution mechanisms should be considered in conjunction with communication tools.

## Open Questions

*   What specific notifications (email, push) should be triggered by order status changes?
*   How will disputes or issues arising from communication be escalated or mediated?
*   Should there be a time limit for Locals to update status or respond to messages?
