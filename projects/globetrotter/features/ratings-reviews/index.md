## Overview

This feature allows both Shoppers and Locals to submit ratings and written reviews for each other upon the completion of a transaction. The aim is to create a self-regulating community where good behavior is rewarded and poor performance is identified, contributing to a safer and more reliable marketplace.

## Design Constraints

*   Rating and review submission must be simple and intuitive.
*   Reviews should only be possible after a completed transaction.
*   Users should have the ability to report inappropriate reviews.
*   Ratings and review data should be prominently displayed on user profiles.
*   System should prevent unfair or retaliatory reviews where possible (e.g., blind reviews until both parties submit).

## Open Questions

*   Should reviews be anonymous until both parties submit, or fully transparent immediately?
*   What is the process for moderating or removing inappropriate/false reviews?
*   How will a user's overall rating be calculated (e.g., simple average, weighted average, decaying average)?
*   Should there be a time limit within which reviews can be submitted after an order completes?
