## Overview

This feature covers the entire lifecycle of an item request, from a Shopper initiating a request for a specific item to Locals discovering these requests and submitting their offers. It includes functionalities for describing items, setting budgets, and transparently outlining costs from the Local's side.

## Design Constraints

*   The item request form must be clear and guide Shoppers to provide necessary details without being overly complex.
*   Locals should have robust tools to search and filter requests relevant to their current location or travel plans.
*   Bid submission should be straightforward, with clear fields for all cost components.
*   Support for attaching images to item requests is essential.

## Open Questions

*   What kind of item categories should be pre-defined to help structure requests?
*   How will currency conversion be handled for item budgets and bids across different regions?
*   What mechanisms will be in place to prevent spam or inappropriate item requests/bids?
